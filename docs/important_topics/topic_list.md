This list contains cybersecurity-related topics that are easy to understand, especially for beginners. Each topic is selected in a simple way so that anyone can learn it without confusion. The topics are arranged clearly, making it easier to understand each concept step by step. This helps learners build a strong foundation in cybersecurity before moving on to more advanced topics.

## **Web Attacks**

### OWASP Top 10
OWASP Top 10 is important because it lists the most common and serious web application security risks. It helps developers, testers, and security professionals understand the biggest security problems and learn how to prevent them. By following the OWASP Top 10, organizations can build safer and more secure web applications.

### XSS (Cross-Site Scripting)
XSS (Cross-Site Scripting) is used by attackers to put harmful code into a website. When users open that website, the code runs in their browser. This can let attackers steal data like cookies, sessions, or personal information and sometimes change what the user sees on the page.

### CSRF (Cross Site Request Forgery)
Cross-Site Request Forgery (CSRF or XSRF) is a web vulnerability that tricks a user into unknowingly executing unwanted actions on a web application in which they are currently authenticated

## **SOC & Operations**

### SOC (Security Operations Center)
A SOC (Security Operations Center) is important because it continuously monitors and protects an organization’s systems from cyber threats. It helps detect attacks early, responds to security incidents quickly, and reduces damage. A SOC also improves overall security by analyzing threats and making systems more safe and reliable.

### SIME or SOAR
SIEM (Security Information and Event Management) and SOAR (Security Orchestration, Automation, and Response) are important because they help security teams detect and respond to cyber threats more efficiently. SIEM collects and analyzes security logs from different systems to identify suspicious activities. SOAR automates security tasks and speeds up incident response, helping organizations reduce response time and improve overall security.


### EDR or XDR
EDR (Endpoint Detection and Response) and XDR (Extended Detection and Response) are important because they help detect, investigate, and stop cyber threats quickly. EDR protects individual devices like laptops and computers, while XDR collects and analyzes security data from multiple sources, such as endpoints, networks, email, and cloud services. This helps security teams find attacks faster and respond more effectively.

### UKC (Unified Kill Chain)
Unified Kill Chain (UKC) is important because it helps security teams understand how cyber attacks happen from start to finish. It combines different attack models to show every stage of an attack, making it easier to detect, stop, and respond to threats. By using UKC, organizations can improve their security and reduce the risk of successful cyber attacks.

## **Threat & Analysis**

### IOC (Indicator of Compromise)
IOCs (Indicators of Compromise) are important because they help identify signs of a cyber attack or security breach. These can include unusual files, IP addresses, or system behavior. By using IOCs, security teams can quickly detect threats, investigate incidents, and stop attacks before they cause more damage.

### TI (Threat Intelligence)
Threat Intelligence (TI) is important because it provides information about current and future cyber threats. It helps security teams understand attackers, their methods, and possible targets. With this knowledge, organizations can prevent attacks early, improve security defenses, and make better decisions to protect their systems and data.

### CTI (Cyber Threat Intelligence)
CTI (Cyber Threat Intelligence) is important because it helps organizations understand cyber threats in detail. It collects and analyzes information about attackers, their tools, and their methods. This helps security teams detect attacks early, prevent future threats, and respond quickly to incidents. CTI makes an organization’s security stronger and more proactive.

### TTP (Tactics, Techniques and Procedures)
TTPs are important because they describe how cyber attackers plan and carry out their attacks. They show the attacker’s behavior step by step. By studying TTPs, security teams can better understand attack patterns, detect threats early, and improve their defense against future attacks.

### APT (Advanced Persistent Threat)
APTs are important because they represent highly skilled and long-term cyber attacks, usually carried out by organized groups. These attackers stay hidden in a system for a long time to steal data or cause damage. Understanding APTs helps security teams detect slow and hidden attacks, improve monitoring, and strengthen overall cybersecurity defense.

### CAR (Cyber Analytics Repository)
CAR (Cyber Analytics Repository) is important because it provides a collection of security analytics and detection rules. It helps security teams identify cyber threats by using ready-made patterns and methods for detecting attacks. CAR makes threat detection faster, improves analysis, and helps organizations strengthen their cybersecurity defenses.

## **Metrics**

### MTTD (Mean Time to Detect)
MTTD is important because it shows how quickly an organization can detect a cyber attack or security issue. A lower MTTD means threats are found faster, which helps reduce damage. It is a key measure to improve monitoring, strengthen security systems, and respond to attacks in time.

### MTTA (Mean Time to Acknowledge)
MTTA is important because it measures how quickly a security team notices and accepts an alert after it is triggered. A low MTTA means the team is responding faster, which helps reduce risk and avoid delays in handling cyber threats. It improves incident response efficiency and overall security performance.

### MTTR (Mean Time to Response)
MTTR is important because it measures how quickly a security team responds to and fixes a cyber incident after it is detected. A lower MTTR means threats are handled faster, which reduces damage and downtime. It helps improve incident response speed and overall cybersecurity effectiveness.

## **Detection Systems**

### IDS (Intrusion Detection System)
IDS is important because it monitors network or system activity and detects suspicious behavior or possible cyber attacks. It sends alerts to security teams so they can investigate and take action. IDS helps in early threat detection and improves overall network security.

### IPS (Intrusion Prevention System)
IPS is important because it not only detects cyber threats but also blocks them in real time. It monitors network traffic and automatically stops malicious activity before it can harm systems or data. This helps improve security and reduce the risk of attacks.

### HIDS (Host Intrusion Detection System)
HIDS is important because it monitors a single computer or host for suspicious activity. It checks system files, logs, and processes to detect possible attacks. If it finds anything unusual, it sends alerts to security teams so they can take action quickly. This helps protect individual systems from threats.

### HIPS (Host Intrusion Prevention System)
HIPS is important because it protects a single computer or host by detecting and blocking malicious activities in real time. It monitors system behavior, files, and processes, and stops harmful actions before they can damage the system. This helps improve endpoint security and prevent attacks.

### NIPS (Network Intrusion Prevention System)
NIPS is important because it protects an entire network by monitoring all incoming and outgoing traffic. It detects malicious activity and automatically blocks attacks in real time. This helps prevent unauthorized access, malware, and other network-based threats.

### WIPS (Wireless Intrusion Prevention System)
WIPS is important because it protects wireless networks from unauthorized access and attacks. It continuously monitors Wi-Fi traffic to detect rogue access points, suspicious devices, and hacking attempts. When it finds a threat, it can block it to keep the wireless network secure.

### NBA (Network Behaviours Analysis)
NBA is important because it monitors and analyzes network traffic patterns to find unusual or abnormal behavior. It helps detect hidden threats like malware, insider attacks, or slow cyber attacks that other tools may miss. This improves early threat detection and overall network security.
